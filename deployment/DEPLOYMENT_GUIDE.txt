## 🚀 Complete Deployment Steps

### STEP 1: Setup MongoDB Database (5 minutes)

1. Go to https://cloud.mongodb.com
2. Sign up for free account
3. Create new cluster (M0 Free tier)
4. Create database user:
   - Username: edmarg_user
   - Password: [Generate strong password]
   - Role: Read and write to any database
5. Network Access → Add IP: 0.0.0.0/0 (Allow from anywhere)
6. Get connection string:
   - Click "Connect" → "Connect your application"
   - Copy: mongodb+srv://edmarg_user:<password>@cluster.mongodb.net/edmarg
   - Replace <password> with your actual password
7. Save this connection string - you'll need it!

---

### STEP 2: Deploy Backend on Render (10 minutes)

1. **Go to Render**
   - Visit: https://render.com
   - Sign up with GitHub/Email

2. **Create Web Service**
   - Click "New +" → "Web Service"
   - Choose "Deploy from Git" OR "Upload code manually"

3. **Upload Backend Code** (if manual)
   - Extract edmarg-backend.zip
   - Upload the folder

4. **Configure Service**
   ```
   Name: edmarg-backend
   Environment: Node
   Build Command: npm install
   Start Command: npm start
   Plan: Free
   ```

5. **Add Environment Variables**
   Click "Environment" tab and add:
   ```
   MONGODB_URI=mongodb+srv://edmarg_user:YOUR_PASSWORD@cluster.mongodb.net/edmarg
   JWT_SECRET=your-super-secret-key-at-least-32-characters-long-random-string
   PORT=5000
   NODE_ENV=production
   FRONTEND_URL=https://yourdomain.com
   ```

   **Generate JWT Secret:**
   ```bash
   # On Mac/Linux terminal:
   openssl rand -base64 32
   
   # Or use any random 32+ character string
   ```

6. **Deploy**
   - Click "Create Web Service"
   - Wait 2-3 minutes for deployment
   - Copy your backend URL: `https://edmarg-backend-xxxx.onrender.com`
   - **SAVE THIS URL!**

7. **Seed Database**
   - Go to your service → "Shell" tab
   - Run these commands:
   ```bash
   npm run seed:admin
   npm run seed:assessments
   ```
   - This creates admin user and sample data

8. **Test Backend**
   - Visit: `https://your-backend-url.onrender.com/api/v1/health`
   - Should see: `{"status":"ok"}`

---

### STEP 3: Deploy Frontend on Hostinger (5 minutes)

1. **Login to Hostinger**
   - Go to: https://hpanel.hostinger.com
   - Login with your credentials

2. **Access File Manager**
   - Click "Websites"
   - Select your domain
   - Click "File Manager"

3. **Prepare Directory**
   - Navigate to `public_html` folder
   - **DELETE** all existing files (index.html, etc.)
   - Make sure folder is empty

4. **Upload Files**
   - Click "Upload" button (top right)
   - Select `edmarg-hostinger.zip`
   - Wait for upload to complete

5. **Extract Files**
   - Right-click on `edmarg-hostinger.zip`
   - Click "Extract"
   - Wait for extraction

6. **Move Files to Root**
   - Open the extracted folder
   - Select ALL files (Ctrl+A or Cmd+A)
   - Click "Move"
   - Move to: `/public_html/`
   - Confirm

7. **Clean Up**
   - Delete `edmarg-hostinger.zip`
   - Delete empty extracted folder

8. **Verify Files**
   Make sure these exist in `public_html`:
   - index.html
   - .htaccess
   - _next/ folder
   - favicon.ico
   - Other folders (admin/, student/, mentor/, etc.)

9. **Set Permissions**
   - Select all files
   - Right-click → "Permissions"
   - Files: 644
   - Folders: 755

### STEP 4: Update Frontend Configuration (IMPORTANT!)

**You need to rebuild frontend with your backend URL:**

1. **On your local machine:**
   ```bash
   cd /Users/bishtshivansugmail.com/Desktop/EdMarg/frontend
   ```

2. **Update .env.production:**
   ```env
   NEXT_PUBLIC_API_BASE_URL=https://your-backend-url.onrender.com/api/v1
   NEXT_PUBLIC_BACKEND_URL=https://your-backend-url.onrender.com
   NEXT_PUBLIC_APP_NAME=EdMarg
   NEXT_PUBLIC_APP_URL=https://yourdomain.com
   NEXT_PUBLIC_ENVIRONMENT=production
   ```

3. **Rebuild:**
   ```bash
   npm run build
   cd out
   zip -r edmarg-hostinger-updated.zip .
   ```

4. **Re-upload to Hostinger:**
   - Delete old files from public_html
   - Upload new zip
   - Extract and move files

## 🔧 Troubleshooting

### Issue: Blank Page
**Symptoms:** Website shows blank white page

**Solutions:**
1. Check browser console (F12) for errors
2. Verify .htaccess file exists in public_html
3. Check file permissions (644 for files, 755 for folders)
4. Clear browser cache (Ctrl+Shift+R)

### Issue: API Connection Failed
**Symptoms:** "Network Error" or "Failed to fetch"

**Solutions:**
1. Verify backend is running on Render
2. Check backend URL in frontend .env.production
3. Verify CORS settings in backend
4. Check FRONTEND_URL in backend environment variables
5. Test backend directly: `https://your-backend-url.onrender.com/api/v1/health`

### Issue: 404 on Page Refresh
**Symptoms:** Refreshing page shows 404 error

**Solutions:**
1. Verify .htaccess file exists
2. Check .htaccess content (should have rewrite rules)
3. Contact Hostinger to enable mod_rewrite

### Issue: Login Not Working
**Symptoms:** Login fails or shows errors

**Solutions:**
1. Check MongoDB connection in backend logs
2. Verify JWT_SECRET is set in backend
3. Check browser cookies are enabled
4. Clear browser cookies and try again

### Issue: Images Not Loading
**Symptoms:** Broken image icons

**Solutions:**
1. Check file permissions
2. Verify image paths are correct
3. Check if images were uploaded

### Issue: Backend Sleeping (Render Free Tier)
**Symptoms:** First request takes 30+ seconds

**Solutions:**
1. This is normal for Render free tier
2. Backend sleeps after 15 minutes of inactivity
3. Use cron-job.org to ping every 10 minutes:
   - Create job at https://cron-job.org
   - URL: `https://your-backend-url.onrender.com/api/v1/health`
   - Schedule: Every 10 minutes
