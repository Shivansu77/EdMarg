# AI Tools Usage Documentation

This document outlines how AI tools were utilized during the development of the EdMarg platform, including specific prompts, errors generated by the AI, and how they were corrected.

## AI Tools Used

1. **GitHub Copilot**: Used continuously in the IDE for code completion, generating boilerplate Express routes, Mongoose schemas, and React components.
2. **ChatGPT (GPT-4)**: Used for architecture planning, designing the career assessment scoring algorithm, and writing complex aggregation pipelines in MongoDB.
3. **Claude (Anthropic)**: Used for writing documentation, analyzing system design decisions, and generating seed scripts.

## Key Prompts Used

- *"Design a MongoDB schema for an assessment template that allows dynamic question types (text, multiple choice, rating) and a corresponding schema to store student responses."*
- *"Write an Express service that calculates a student's career fit based on 8 tags (logic, tech, social, etc.) and compares them against 10 predefined career profiles using weighted scoring."*
- *"Create a Socket.io implementation for a Next.js frontend that handles private messaging between two users, including read receipts and online status indicators."*

## AI Errors & Corrections

### Case 1: MongoDB `$lookup` with Array of ObjectIds
**What the AI produced:**
When generating a query to fetch an assessment assignment along with the populated student profiles, the AI generated a standard `.populate('assignedTo')`. However, we needed a complex aggregation pipeline to filter students based on specific criteria within the array. The AI wrote:
```javascript
{ $lookup: { from: 'users', localField: 'assignedTo', foreignField: '_id', as: 'students' } }
```

**How it was caught:**
The query returned an empty `students` array. Testing the pipeline in MongoDB Compass revealed that `assignedTo` was an array of ObjectIds, but the AI assumed it was a single ObjectId.

**What was changed:**
I modified the pipeline to use `$unwind` before the lookup, or used the more modern `$lookup` syntax that natively supports arrays:
```javascript
// Corrected to use mongoose populate which handles arrays natively,
// or using $in within a custom $lookup pipeline
```

### Case 2: Asynchronous Loop Pitfall
**What the AI produced:**
When writing a script to seed the database with multiple templates and assignments, the AI wrote a `forEach` loop with asynchronous operations inside:
```javascript
templates.forEach(async (template) => {
  await AssessmentTemplate.create(template);
});
console.log('Seeding complete');
```

**How it was caught:**
The script printed 'Seeding complete' immediately and exited before the database operations finished, resulting in an empty database.

**What was changed:**
I replaced the `forEach` loop with a `for...of` loop to ensure the asynchronous operations were properly awaited before the script terminated:
```javascript
for (const template of templates) {
  await AssessmentTemplate.create(template);
}
console.log('Seeding complete');
```

### Case 3: React State Stale Closure
**What the AI produced:**
In the frontend assessment taking component, the AI generated a socket event listener inside a `useEffect` that updated the current answers state:
```javascript
useEffect(() => {
  socket.on('mentor_suggestion', (data) => {
    setAnswers({ ...answers, newSuggestion: data });
  });
}, []);
```

**How it was caught:**
During testing, when a mentor sent a suggestion, previous answers entered by the student were wiped out. The AI had created a stale closure, locking `answers` to its initial empty state when the component mounted.

**What was changed:**
I updated the state setter to use the functional update pattern, ensuring it always receives the most recent state:
```javascript
useEffect(() => {
  socket.on('mentor_suggestion', (data) => {
    setAnswers(prev => ({ ...prev, newSuggestion: data }));
  });
}, []);
```
